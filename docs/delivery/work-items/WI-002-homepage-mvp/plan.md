# WI-002 计划

## 实施方案

### 1. 产品策略

- 首页服务“单人自用的收藏回找”主任务，而不是内容发现
- 置顶属于个人偏好层，不进入站点内容源
- 置顶通过同列表优先排序体现，不新增独立置顶区块

### 2. 内容源落地

- 新增受 `DATA_MODEL` 约束的站点类型定义
- 在仓库内建立 `data/sites.ts` 作为首页内容源
- 先准备一组覆盖多个分类、标签和排序场景的样例数据

### 3. 纯逻辑先行

- 抽离搜索、分类筛选、状态过滤、排序逻辑
- 抽离浏览器本地置顶的读写逻辑
- 保证逻辑层不依赖 UI，便于后续验证与维护

### 4. 首页结构实现

- 用首页承载标题、说明、搜索区、分类区、结果区、空状态区
- 将卡片、搜索栏、分类栏拆为独立组件
- 控制客户端边界，只把即时交互部分放入客户端组件

### 5. 体验策略

- 搜索覆盖名称、描述、标签
- 分类与搜索叠加生效
- 当前视图可见的置顶站点在结果区前置
- 置顶站点通过浏览器 `localStorage` 持久化
- 置顶不单独拆区，卡片布局保持一致
- 卡片信息强调“用途判断”而不是装饰

### 6. 验证与回写

- 先做最小必要自检，确认首页主链路可用
- 若实现细节改变了当前资产定义，回写到相应权威文档

## 建议文件落点

- `app/page.tsx`
- `components/navigation/`
- `data/sites.ts`
- `lib/site-index.ts`
- `lib/pinned-sites.ts`
- `types/site.ts`

## 执行顺序

1. 站点类型与数据
2. 过滤/排序与本地偏好逻辑
3. 展示组件与首页组装
4. 验证与文档回写
