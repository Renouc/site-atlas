# WI-004 简述

## 标题

全项目文档同步与状态校正

## 背景

WI-002 与 WI-003 的实现已经落地，但仓库内仍存在多处文档滞后：`README` 与 `ROADMAP` 状态未更新，`manifest` 未登记 `WI-003`，产品、设计、工程文档也未完整覆盖主题切换、搜索快捷键、favicon 代理、域名衍生展示等当前实现事实。

## 目标

- 基于当前仓库实现校正文档状态
- 补齐遗漏的工作单登记与当前活动工作单
- 回写受影响的产品、设计、工程与运维权威资产
- 将“实现已变、文档未跟”的偏差收敛到当前治理体系内

## 边界

### 本工作单包含

- 创建文档同步工作单并更新 `manifest`
- 同步 `README.md`、`docs/delivery/ROADMAP.md` 等项目状态文档
- 同步 `PRD`、`IA`、`UX_RULES`、`HOME`、`ARCHITECTURE`、`DATA_MODEL`
- 必要时补充维护与内容流程说明

### 本工作单不包含

- 任何功能代码改动
- 新增产品功能或技术方案
- 引入新的内容源、后台系统或数据模型扩展
- 重写既有工作单的历史结论

## 关键路径

- 先以仓库现状为事实依据梳理主链路与已落地能力
- 再更新工作单、状态文档与权威资产
- 最后对照当前实现做一致性自检，记录同步结果
