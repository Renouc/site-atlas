# WI-005 简述

## 标题

开发站点补充与轻量筛选

## 背景

当前仓库已具备站点导航与维护规范，但 `data/sites.ts` 仍以手工维护为主。用户本轮明确提出补充一批前端、Node.js、桌面端、多端与 AI 工作流相关的官方文档站点，并希望对不建议纳入的候选给出单独原因说明。

## 目标

- 将本轮确认的高价值官方文档站点补充进 `data/sites.ts`
- 在不扩展数据模型的前提下完成 URL 归一化、分类、标签与描述补全
- 对存在保留意见的候选给出简洁说明，避免只做机械追加

## 边界

### 本工作单包含

- 校验候选链接是否为官方入口或更合适的规范化地址
- 复用现有分类与标签体系补充站点数据
- 记录本轮站点增补结果与筛选说明

### 本工作单不包含

- 新增分类、字段或站点数据模型变更
- 引入自动导入、后台维护系统或额外内容源
- 调整首页交互、偏好层逻辑或渲染结构

## 关键路径

- 先按维护规范做去重与官方链接归一化
- 再写入 `data/sites.ts`
- 最后记录验收结果与保留意见
