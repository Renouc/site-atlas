# 内容工作流

## 1. 需求进入

- 非琐碎需求先创建或更新工作单
- 识别本次任务属于产品、设计、工程还是运营变更
- 若发现代码已先于文档发生变化，先补建文档同步工作单，再集中回写资产

## 2. 读取资产

- 按 `AGENTS.md` 与 `routing.yaml` 要求读取权威资产
- 若缺少资产，先补资产再进入实现

## 3. 执行变更

- 先修改权威定义，再修改实现
- 若仅是实现细节调整且不影响权威定义，可直接修改实现
- 若本轮目标是文档同步，应以当前仓库实现为事实依据，集中回写所有受影响资产

## 4. 回写与验收

- 对照工作单 `acceptance.md` 自检
- 在 `review.md` 记录完成情况与偏差
- 若本轮变更影响路线图、维护方式或当前活动工作单，同步更新相关状态文档与 `manifest`

## 5. 关闭条件

- 当前工作单验收通过
- 权威资产与实现一致
- 下一步建议已明确
