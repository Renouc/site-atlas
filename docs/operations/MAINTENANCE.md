# 维护手册

## 1. 新增站点前检查

- 该站点是否确实属于高频或高价值资源
- 是否已有等价站点条目
- 是否能用现有分类与标签表达
- 是否仅需要补充备注，而不是扩展模型字段

## 2. 新增站点最小要求

- 必须填写名称
- 必须填写可访问 URL
- 必须填写一句话用途描述
- 必须归属一个受控分类
- 状态必须明确
- 如无额外上下文，`note` 可留空
- 无需单独维护域名或 favicon 字段，展示时从 `url` 派生

## 3. 分类维护规则

- 先复用现有分类
- 只有当一组站点持续增长且现有分类无法清晰承载时，才新增分类
- 分类变更需同步更新 `IA` 与 `DATA_MODEL`

## 4. 标签维护规则

- 优先复用已有标签
- 避免语义重复标签，例如“文档”和“docs”并存
- 标签新增前先判断是否真实提升检索效率

## 5. 清理规则

- 长期失效或不再需要的站点，先调整为 `archived`
- 不直接物理删除，除非确认不再需要保留历史上下文

## 6. 本地偏好维护规则

- 置顶是个人浏览器偏好，不在内容源中维护
- 主题同样属于浏览器本地偏好，不进入站点内容模型
- 禁止在站点数据中新增 `pinned`、`theme`、`favorite` 之类字段承载个人习惯
- 若需要调整置顶或主题行为，优先修改偏好层逻辑与相关权威资产，而不是修改内容模型

## 7. 变更后检查

- 展示字段是否完整
- 分类与标签是否符合规则
- URL 是否可用且可派生正确域名
- 权威资产是否同步更新
