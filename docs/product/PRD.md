# PRD

## 1. 当前阶段

- 当前阶段目标：在知识治理基座上完成可日常使用的首页导航，服务单人自用收藏管理场景
- 当前阶段完成标志：首页支持搜索、分类、本地置顶、主题切换与站点卡片浏览，并保持 docs 与实现一致

## 2. MVP 目标

- 让用户能在一个首页中快速回找和打开已收藏站点
- 让每个站点具备结构化语义信息
- 让常用站点拥有浏览器本地持久化的置顶能力
- 提供轻量的深浅色主题切换，保证不同环境下的可读性
- 让后续新增站点和调整分类有稳定规则
- 让 AI 能根据权威资产继续推进实现而不跑偏

## 3. MVP 范围

### 包含

- 首页导航页
- 站点卡片列表
- 搜索框与最小重置操作
- 分类筛选
- 标签展示
- 站点置顶与取消置顶
- 当前列表内的置顶优先排序
- 深浅色主题切换
- 响应式布局
- 使用仓库内类型化数据作为内容源
- 使用浏览器 `localStorage` 持久化置顶偏好
- 使用浏览器本地偏好与系统主题完成主题初始化

### 不包含

- 登录与权限
- 多用户协作
- 数据库
- 复杂后台管理界面
- 推荐算法
- 访问统计
- 跨设备同步置顶或主题偏好

## 4. 功能需求

### FR-1 首页展示

- 首页展示产品标题、一句话说明与主题切换入口
- 首页展示站点卡片集合
- 当前视图下存在可见置顶站点时，这些站点在同一结果列表内优先展示
- 置顶不形成独立内容区块，不打断按分类浏览的主路径
- 用户无需进入二级页面即可理解站点用途

### FR-2 搜索

- 支持基于站点名称、描述、标签的关键词搜索
- 搜索结果应即时反馈
- 用户可通过最小重置操作快速回到初始状态

### FR-3 分类筛选

- 支持按分类进行筛选
- 分类切换后结果应与搜索条件叠加生效
- 分类仅承载站点语义领域，不包含“常用”这类个人偏好维度

### FR-4 本地置顶偏好

- 用户可将站点置顶或取消置顶
- 置顶偏好保存在当前浏览器的 `localStorage`
- 置顶偏好基于站点 `id` 关联，而不是写入站点内容源
- 置顶仅影响当前视图结果列表中的排序优先级，不改变站点内容源与卡片结构

### FR-5 条目语义

- 每个站点必须至少包含名称、URL、描述、分类、标签、状态
- 备注字段仅在确有补充价值时填写
- 禁止只有标题和 URL 的无语义条目进入主数据源

### FR-6 可维护性

- 新增站点时有明确字段规范
- 分类与标签新增有约束，不可无序增长
- 置顶与主题均属于浏览器本地偏好，不进入站点内容模型，避免内容层被个人使用习惯污染

### FR-7 主题切换

- 用户可在首页切换深色/浅色主题
- 首次访问时，若无本地主题偏好，页面应跟随系统主题
- 用户主动切换后，应在当前浏览器中持久保留主题选择
- 主题切换不影响站点内容、筛选条件与排序逻辑

## 5. 非功能需求

- 页面结构简洁，首屏即可使用
- 默认性能足够轻量，不依赖后端服务
- 代码结构与数据结构可读，便于长期维护
- 客户端交互边界最小化，仅在需要浏览器能力和即时交互时使用客户端组件

## 6. 验收标准

- 用户可以在首页完成搜索与分类筛选
- 用户可以在当前浏览器置顶/取消置顶常用站点
- 刷新页面后置顶状态仍然存在
- 置顶站点在当前结果列表内优先出现
- 页面不存在独立置顶区块或重复展示
- 站点卡片信息足够支撑快速判断是否要打开
- 用户首次访问时可根据本地偏好或系统主题获得正确主题
- 用户点击主题按钮后页面立即切换深浅色，刷新后仍保留选择
- 置顶仅通过轻量状态标记区分，不制造重复扫描负担
- 数据模型能覆盖当前 MVP 所需字段
- 文档资产与实现保持一致

## 7. 风险与约束

- 分类过多会削弱筛选效率
- 字段过多会增加维护成本
- 过早引入后台会显著增加复杂度
- 将个人偏好写入内容源会破坏内容层与偏好层边界
- 将视觉偏好扩展成复杂配置系统会偏离当前 MVP 目标

## 8. 后续候选需求

- 浏览器书签导入
- 手动编辑界面
- 最近新增/最近访问
