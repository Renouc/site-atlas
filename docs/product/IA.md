# 信息架构

## 1. 导航结构

V1 仅保留一个主页面：首页导航页。所有高频行为都应在首页完成，不依赖深层页面跳转。首页的主要任务是“回找与打开已收藏站点”，而不是“发现新内容”。

## 2. 首页信息区块

- 顶部：产品标题与简要说明
- 搜索区：关键字输入
- 分类区：分类切换
- 置顶区：高频常用站点，仅在存在可见置顶时展示
- 结果区：所有符合条件的站点卡片
- 空状态区：无结果时的引导说明

## 3. 分类体系

### 当前建议分类

- 常用
- 开发
- AI / LLM
- 设计
- 工具
- 学习
- 资讯
- 其他

### 分类规则

- 分类用于一级粗筛，数量应控制在用户可快速扫描的范围内
- 不允许按临时主题无限扩张分类
- 若某类站点规模过小，优先使用标签而不是新增分类

## 4. 标签体系

### 标签职责

- 标签用于细粒度语义召回
- 支持跨分类表达，例如“文档”“社区”“部署”“搜索”“效率”

### 标签规则

- 标签词汇尽量短、稳定、可复用
- 优先复用既有标签，不轻易创建新标签
- 标签数量以能提升查找效率为准，不追求覆盖一切信息

## 5. 站点卡片信息层级

### 必须展示

- 名称
- 描述
- 分类
- 标签

### 可选展示

- 推荐理由/备注
- 置顶操作入口

## 6. 内容层与偏好层

- 内容层：站点名称、URL、描述、分类、标签、状态等稳定信息
- 偏好层：是否置顶等个人使用习惯信息
- V1 中，偏好层使用浏览器本地存储承载，不进入仓库内站点内容源

## 7. 结果组织规则

- 若当前筛选条件下存在置顶站点，则先展示置顶区
- 已出现在置顶区的站点，不在结果区重复展示
- 若当前筛选条件下不存在置顶站点，则直接展示结果区
- 若无任何匹配结果，则仅展示空状态区

## 8. 信息查找路径

- 已知站点名：直接搜索
- 只记得用途：搜索描述或标签
- 只知道领域：先选分类再扫描卡片
- 常用高频：直接从置顶区进入
