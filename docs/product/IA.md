# 信息架构

## 1. 导航结构

- 当前 MVP 采用单页首页结构
- 主链路为：进入首页 → 搜索或切换分类 → 扫描卡片 → 打开目标站点
- 主题切换属于页头辅助控制，不改变信息层级与主任务路径

## 2. 首页信息区块

- 顶部：产品标识、一句话说明、主题切换入口
- 搜索区：关键词输入与快捷聚焦入口
- 分类区：固定分类切换
- 结果区：所有符合条件的站点卡片，可见置顶项在同一列表内前置
- 空状态区：无结果时的引导说明

## 3. 分类体系

### 当前建议分类

- 开发
- AI / LLM
- 设计
- 工具
- 学习
- 资讯
- 其他

### 分类规则

- 分类用于一级粗筛，数量应控制在用户可快速扫描的范围内
- “常用”不属于分类，而属于个人偏好层
- 不允许按临时主题无限扩张分类
- 若某类站点规模过小，优先使用标签而不是新增分类

## 4. 标签体系

### 标签职责

- 标签用于细粒度语义召回
- 支持跨分类表达，例如“文档”“社区”“部署”“搜索”“效率”

### 标签规则

- 标签词汇尽量短、稳定、可复用
- 优先复用既有标签，不轻易创建新标签
- 标签数量以能提升查找效率为准，不追求覆盖一切信息

## 5. 站点卡片信息层级

### 必须展示

- 站点识别层：favicon 或首字母头像、名称、域名
- 描述
- 分类
- 标签

### 可选展示

- 推荐理由/备注
- 置顶操作入口
- 轻量置顶状态标记
- 新标签页打开提示

## 6. 内容层与偏好层

- 内容层：站点名称、URL、描述、分类、标签、状态、备注等稳定信息
- 偏好层：是否置顶、主题选择等个人使用习惯信息
- V1 中，偏好层使用浏览器本地存储承载，不进入仓库内站点内容源

## 7. 结果组织规则

- 若当前筛选条件下存在置顶站点，则在结果区同一列表内优先展示
- 置顶站点保持与普通站点一致的卡片结构，不单独分组
- 若当前筛选条件下不存在可见置顶站点，则结果区按默认排序展示
- 若无任何匹配结果，则仅展示空状态区

## 8. 信息查找路径

- 已知站点名：直接搜索
- 只记得用途：搜索描述或标签
- 只知道领域：先选分类再扫描卡片
- 高频常用：进入当前视图后优先扫描列表前部的置顶项
- 夜间或弱光场景：通过页头主题切换保持可读性
