# 项目章程

## 项目定义

Site Atlas 是一个面向个人使用场景的高频站点导航项目，用于替代浏览器收藏夹中“只存不管、越来越难找”的使用方式。它不是通用书签仓库，而是一个强调高频访问效率、站点语义描述、分类检索和持续维护能力的导航产品。

## 目标用户

- 链接收藏量持续增长、回忆成本高的个人用户
- 高频使用开发、AI、设计、学习类站点的知识工作者
- 希望通过代码仓库持续维护个人导航资产的使用者

## 核心问题

- 浏览器收藏夹更适合“保存”，不适合“理解与回找”
- 链接缺乏结构化描述，难以通过语义快速回忆
- 分类和标签体系缺失，新增越多越混乱
- 长期维护没有规则，内容资产容易失真

## 成功标准

- 用户能在首页通过搜索或筛选快速定位目标站点
- 每个站点都具备清晰语义信息，而不只是 URL
- 新增、删除、重分类站点时，有明确维护规则
- AI 在后续迭代中能基于同一套资产稳定执行，不频繁跑偏

## 非目标

- 第一阶段不做多用户系统
- 第一阶段不做登录、权限、数据库后台
- 第一阶段不做复杂的内容编辑器
- 第一阶段不做访问统计与推荐算法

## 当前阶段定义

- 当前阶段目标：完成知识治理基座，并为首页 MVP 提供可执行的产品、设计、工程资产
- 当前阶段完成标志：知识体系可驱动后续首页实现，AI 能按固定入口和优先级执行
