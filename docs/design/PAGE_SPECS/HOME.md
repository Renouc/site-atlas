# 首页规格

## 1. 页面目标

- 在单页内完成站点回找、筛选、打开与常用前置
- 让用户在不进入二级页面的情况下完成大部分判断
- 在不同环境光线下保持稳定可读性

## 2. 页面结构

### 2.1 页头

- 产品标识
- 一句话说明
- 主题切换图标入口

### 2.2 搜索区

- 搜索输入框
- `/` 快捷聚焦提示
- 当存在搜索词或分类筛选时显示最小重置操作

### 2.3 分类区

- “全部”选项
- 固定分类列表

### 2.4 结果区

- 直接展示经过搜索与筛选后的卡片列表
- 当前视图下可见的置顶站点在同一列表内按本地置顶顺序优先展示
- 置顶站点与普通站点使用相同卡片布局，不单独拆分区块
- 不额外渲染结果标题、数量或解释性状态文案
- 单张卡片包含：站点图标或首字母头像、名称、外链提示、域名、分类、描述、可选备注、标签与置顶操作

### 2.5 空状态区

- 当无结果时展示
- 引导用户修改搜索词或切换分类

## 3. 页面行为

- 输入搜索词后即时过滤
- 当焦点不在输入态元素时，按 `/` 聚焦搜索框
- 搜索框聚焦时按 `Escape`，有内容则清空，无内容则失焦
- 点击分类后叠加分类条件
- 点击站点卡片以新标签页打开目标站点
- 点击置顶按钮后，当前浏览器立即更新置顶状态并持久化
- 刷新页面后，置顶状态仍保留
- 点击主题切换后，页面立即切换深浅色并保留当前浏览器选择

## 4. 展示优先级

- 当前视图可见的置顶站点优先
- 同为置顶时按本地置顶顺序
- 非置顶站点按显式排序字段
- 仍无排序字段时按名称稳定排序

## 5. 验收口径

- 首页在无数据异常情况下可独立完成主任务
- 搜索、分类、置顶、主题切换、结果展示逻辑相互一致
- 置顶在当前结果列表内优先展示，不拆分独立区块
- 主题初始化、点击切换与刷新保留形成闭环
- 站点域名与图标来自现有 `url` 派生，不要求额外维护字段
- 页面结构与 `PRD`、`IA`、`DATA_MODEL` 一致
