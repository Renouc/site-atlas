# 首页规格

## 1. 页面目标

让用户在一个页面中完成收藏站点的回找、筛选、置顶和跳转，形成比浏览器收藏夹更高效的访问体验。

## 2. 页面结构

### 2.1 页头

- 产品名称
- 一句话说明
- 主题切换图标入口

### 2.2 搜索区

- 搜索输入框
- 最小必要的重置操作

### 2.3 分类区

- “全部”选项
- 固定分类列表

### 2.4 结果区

- 直接展示经过搜索与筛选后的卡片列表
- 当前视图下可见的置顶站点在同一列表内按本地置顶顺序优先展示
- 置顶站点与普通站点使用相同卡片布局，不单独拆分区块
- 不额外渲染结果标题、数量或解释性状态文案
- 非置顶卡片按排序字段、名称等规则稳定展示

### 2.5 空状态区

- 当无结果时展示
- 引导用户修改搜索词或切换分类

## 3. 页面行为

- 输入搜索词后即时过滤
- 点击分类后叠加分类条件
- 点击站点卡片以新标签页打开目标站点
- 点击置顶按钮后，当前浏览器立即更新置顶状态并持久化
- 刷新页面后，置顶状态仍保留

## 4. 展示优先级

- 当前视图可见的置顶站点优先
- 同为置顶时按本地置顶顺序
- 非置顶站点按显式排序字段
- 仍无排序字段时按名称稳定排序

## 5. 验收口径

- 首页在无数据异常情况下可独立完成主任务
- 搜索、分类、置顶、结果展示逻辑相互一致
- 置顶在当前结果列表内优先展示，不拆分独立区块
- 置顶状态不写入站点内容源
- 页面文案克制，不解释实现细节
- 页面结构与 `PRD`、`IA`、`DATA_MODEL` 一致
