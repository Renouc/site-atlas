# 交互规则

## 1. 搜索规则

- 搜索为首页主入口，位置应足够显著
- 搜索范围覆盖名称、描述、标签
- 搜索反馈应即时，无需单独提交按钮
- 当焦点不在输入态元素时，按 `/` 应直接聚焦搜索框
- 搜索框聚焦时按 `Escape`，若已有关键词则清空；若为空则失焦

## 2. 筛选规则

- 分类筛选与搜索条件叠加生效
- 默认分类为“全部”
- 切换分类不应清空搜索词
- “常用”不作为分类选项出现

## 3. 置顶规则

- 置顶用于承载用户最高频访问站点
- 置顶是个人偏好，不属于站点内容事实
- 置顶状态保存在当前浏览器的 `localStorage`
- 置顶只改变当前视图结果列表中的排序优先级
- 置顶站点与普通站点使用同一套卡片布局与信息层级
- 置顶仅通过轻量状态标记与操作入口区分，避免抢占主内容空间
- 若当前视图下不存在可见置顶站点，结果列表按默认排序展示

## 4. 主题规则

- 首次访问时，若无本地主题记录，则跟随系统主题
- 用户点击主题按钮后页面应立即切换
- 用户主动切换后的主题选择保存在当前浏览器
- 主题切换只影响视觉呈现，不改变内容、筛选与排序逻辑

## 5. 卡片规则

- 卡片主要任务是帮助用户快速判断“要不要点开”
- 卡片应优先展示可快速识别站点的信息：图标或首字母、名称、域名
- 描述应保持短句，优先说明用途
- 标签不宜过多，避免视觉噪声
- 备注仅在确有补充价值时展示
- 卡片应提供轻量、低干扰的常用操作入口
- 点击卡片以新标签页打开目标站点

## 6. 空状态规则

- 无搜索结果时，提示用户调整关键词或切换分类
- 不出现“页面空白”的无反馈状态

## 7. 文案规则

- 文案优先表达使用价值，而不是解释实现方式
- 非必要状态、数量与说明不默认展示
- 文案保持短句、克制，避免后台式或说明书式表述

## 8. 响应式规则

- 移动端优先保证搜索和卡片阅读
- 分类切换在窄屏下仍应清晰可操作
- 不依赖 hover 才能获得关键信息

## 9. 本地持久化规则

- 首次访问无本地数据时，默认无置顶，并按系统主题展示
- 读取到失效或不存在的站点 `id` 时自动忽略
- 刷新页面后，当前浏览器中的置顶状态与主题选择均应保留
