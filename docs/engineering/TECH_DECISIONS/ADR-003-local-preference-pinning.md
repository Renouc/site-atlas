# ADR-003：置顶采用浏览器本地偏好层

## 状态

Accepted

## 背景

首页第一版主要服务单人自用的收藏管理场景。用户明确提出，常用站点的置顶应基于浏览器本地持久化能力实现，并且在没有置顶站点时不应占位。此前的数据模型中若直接将 `pinned` 放入站点内容源，会把个人使用习惯与站点事实混在一起。

## 决策

- 站点内容源不包含 `pinned` 字段
- 置顶状态由浏览器 `localStorage` 保存
- 本地偏好仅保存站点 `id` 列表
- 若当前视图下没有可见置顶站点，则不渲染置顶区块
- 已在置顶区展示的站点，不在结果区重复展示

## 结果

### 正面结果

- 内容层与偏好层边界清晰
- 仓库内内容数据保持稳定、可版本化
- 能满足单人自用场景下的个性化习惯
- 无置顶时页面视觉更干净

### 代价

- 置顶偏好只在当前浏览器生效，不跨设备同步
- 服务端首屏无法直接获知本地偏好，需要客户端接管相关交互

## 替代方案

### 方案一：将 `pinned` 写入站点内容源

- 优点：实现简单
- 缺点：个人偏好污染内容模型，不利于长期维护

### 方案二：使用远程存储保存置顶

- 优点：可跨设备同步
- 缺点：显著增加第一版复杂度，不符合当前最小闭环

## 何时重审

- 当项目从单人自用扩展到多设备同步需求
- 当需要将个人偏好跨浏览器、跨设备共享
