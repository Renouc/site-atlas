# ADR-002：V1 使用仓库内类型化内容源

## 状态

Accepted

## 背景

V1 的核心目标是快速形成可用导航首页，并保持后续维护成本低。如果一开始引入数据库、后台或远程 CMS，会显著增加复杂度和变更面，不利于最小闭环。

## 决策

V1 使用仓库内、类型化的内容源作为站点数据单一事实源，推荐承载位置为 `data/sites.ts`。

## 结果

### 正面结果

- 内容结构稳定，可直接受 TypeScript 约束
- 无需后端即可完成 MVP
- 版本控制天然记录内容演进
- 便于 AI 直接读取和修改

### 代价

- 内容维护仍需通过代码仓库进行
- 非技术用户直接维护门槛较高

## 何时重审

- 当站点规模显著增长，手工维护成本明显上升
- 当需要多人协作维护
- 当需要导入、审核、发布等流程能力
