# 架构说明

## 1. 当前架构目标

以最小复杂度承接 V1 首页导航能力，同时为后续内容扩展和局部重构保留清晰边界。

## 2. 承接原则

- 以 App Router 为基础
- 以仓库内类型化数据作为内容源
- 以服务端组装 + 必要客户端交互为默认模式
- 以模块职责清晰为优先，不引入不必要中间层

## 3. 建议模块划分

- `app/page.tsx`：首页入口与页面组装
- `components/navigation/`：搜索栏、分类栏、站点卡片等展示组件
- `data/sites.ts`：站点内容源
- `lib/site-index.ts`：过滤、分组、排序等纯逻辑
- `types/site.ts`：站点数据类型定义

## 4. 数据流

- 静态数据从 `data/sites.ts` 进入
- 首页读取数据并传递给展示层
- 搜索与筛选逻辑保持纯函数，便于测试

## 5. 渲染策略

- 页面内容默认可由服务端读取并输出
- 搜索和分类交互若需即时更新，可在最小范围内使用客户端组件
- 禁止为了简单过滤逻辑而把整个页面不必要地变成客户端组件

## 6. 扩展边界

### 当前不引入

- 数据库
- CMS
- 登录系统

### 后续可扩展

- 书签导入解析
- 本地 JSON / YAML 数据源导入
- 简易维护后台

## 7. 一致性要求

- 实际实现必须遵循 `DATA_MODEL`
- 页面行为必须遵循 `HOME.md` 与 `UX_RULES.md`
- 若架构发生实质变化，必须补 ADR
